Ambient Intelligence at Work

AccessBridge continuously monitors subtle behavioral signals to understand user needs. When struggle patterns emerge, the AI decision engine triggers targeted adaptations without requiring any explicit action.

  • Detects motor impairment via click accuracy and cursor tremor
  • Identifies vision struggles from scroll and zoom behavior
  • Adapts layout complexity based on cognitive load signals
  • All processing runs locally -- no data leaves the browser

Signal → Adaptation Pipeline

Four stages from raw mouse event to applied DOM mutation. Runs entirely inside the Chrome process — the background service worker handles scoring, content scripts do collection and mutation. Nothing leaves the browser for adaptation decisions.

1

Collect

Five passive DOM listeners sample scroll velocity, click accuracy, typing rhythm, cursor path, and dwell time without blocking input.

addEventListener · passive · 5 s buffer
2

Score

Each signal is normalized to 0–1, batched, and fed into an EMA-smoothed weighted composite that produces a live struggle score.

StruggleDetector · 3 s tick · EMA α = 0.3
3

Decide

A rule-based classifier evaluates the score against the user profile and emits one of 16 adaptation types when a confidence threshold is crossed.

DecisionEngine · 16 rules · profile-gated
4

Adapt

The adaptation is routed to SensoryAdapter, CognitiveSimplifier, VoiceCommand, or DomainConnector, which mutates the DOM with a reversible handle.

sendMessage · shadow-CSS vars · reversible

Layered System

Each layer owns a single concern and communicates through a typed message bus. Swap any layer (UI framework, AI provider, signal source) without rewiring the rest.

L1 · UI
React 18 popup + side panel — struggle gauge, toggle grid, profile editor, self-hosted update banner. State mirrored to chrome.storage.local so it survives popup close.
L2 · Content
10+ feature modules injected per tab: SensoryAdapter, CognitiveSimplifier, VoiceCommandSystem, EyeTracker, DwellClick, KeyboardOnlyMode, PredictiveInput, TransliterationController, AIBridge, 6 DomainConnectors.
L3 · Worker
Background service worker hosts StruggleDetector, DecisionEngine, ProfileStore, and the 3-tier AI engine. Persists profile, broadcasts PROFILE_UPDATED to every open tab.
L4 · Browser
Native Chrome APIs only — Web Speech (voice I/O), Shape Detection (FaceDetector for gaze), MutationObserver (dynamic content), chrome.storage / tabs / runtime / downloads. No polyfills, no shims.
L5 · Cloud
Optional and rare — the local AI tier handles ~90% of summarization and simplification. Only long-form or novel prompts reach Gemini Flash or Claude Sonnet, gated by a per-day cost budget.

Monorepo Packages

Three typed packages wired through pnpm workspaces. Every cross-package import goes through an exports map so refactors land atomically.

@accessbridge/core

Types, ProfileStore, StruggleDetector, DecisionEngine, i18n (transliteration rules + language ranges). Zero external runtime dependencies.

@accessbridge/extension

Manifest V3 Chrome extension. React popup + side panel, content scripts, 6 domain connectors, SensoryAdapter, TransliterationController, LanguageDetector.

@accessbridge/ai-engine

Accessibility Challenges We Solve

The lived barriers AccessBridge was built to dissolve — spanning vision, motor, cognitive, linguistic, temporal, and social dimensions of accessibility. Each pairs a real-world problem with the on-device mechanism that addresses it.

Vision

Small text, low contrast, color blindness

Roughly 2.2 billion people live with some form of vision impairment. Yet most websites ship a single 14-16px body size, fixed contrast, and a palette that's invisible to 8% of men with red–green color blindness.

AccessBridge does: detects repeated zooming, slower scroll, and re-reading of the same paragraph, then quietly scales text, deepens contrast, and remaps colors for protanopia / deuteranopia / tritanopia — no site cooperation required.

Engages · Sensory Adapter · Auto Adaptations · Color Correction
Motor

Can't use a mouse or fine-motor control

People with Parkinson's, ALS, severe arthritis, or hand tremor can't reliably click a 16px button or drag a scrollbar. Dedicated assistive hardware (gaze trackers, switch devices) costs $1,500 – $3,500 and needs setup most users never finish.

AccessBridge does: turns any laptop webcam into a gaze cursor (30-second calibration), adds hover-to-click with a visible progress ring, and runs the browser by voice in 28 languages — all with zero extra hardware.

Engages · Eye Tracking · Dwell Click · Voice Commands · Keyboard-Only Mode
Cognitive

Attention difficulties & cognitive overload

People with ADHD, autism, traumatic brain injury, or anxiety get swamped by modern web pages where 70–80% of the pixels are sidebars, ads, notifications, and recommendations — all competing for attention they can't spare.

AccessBridge does: a soft spotlight follows the paragraph being read while everything else dims; ads and sidebars hide on demand; layout simplifies automatically when struggle signals rise — the page gets quieter, not louder.

Engages · Focus Mode · Reading Guide · Distraction Shield · Auto Adaptations
Language

Content & tooling isn't in my language

Over 4.8 billion people speak a non-English primary language. Enterprise apps ship English-first, voice features rarely understand Tamil or Bengali, and Latin→Indic keyboards are a Windows-only pain no one wants to configure.

AccessBridge does: voice recognition in 28 languages (10 Indian first-class + English + 17 global including Portuguese, Russian, Indonesian, Korean, Persian, Thai); Unicode-range page detection sets the voice locale automatically; press Alt+T in any input to type in Latin and watch it become Devanagari / Tamil / Telugu / Kannada.

Engages · Multi-Language Voice · Transliteration · Language Auto-Detect
Temporal

Fatigue & eye strain over long sessions

Mental fatigue, digital eye strain, and migraine thresholds build up across an 8-hour workday. An interface that felt fine at 9 am is hostile at 10 pm — but no website knows or cares what time you started.

AccessBridge does: tracks time-of-day and session length, then progressively simplifies the UI through four calmer levels — hiding optional panels, opening spacing, softening colors, reducing motion — without the user pressing a thing.

Engages · Fatigue-Adaptive UI · Reduced Motion · Contrast Easing
Comprehension

Dense jargon on critical portals

Banking, insurance, healthcare, and legal sites assume every user parses "deductible," "prior authorization," "underwriting," "1.5 Lakh." When you can't, the stakes are real — a missed claim, a wrong premium, a refused treatment.

AccessBridge does: on-device AI rewrites jargon into plain language and summarizes long emails into bullets; 6 domain connectors (Banking, Insurance, Telecom, Retail, Healthcare, Manufacturing) decode terms inline as you browse.

Engages · AI Simplification · AI Summarization · Domain Connectors
Social

The stigma of asking for help

Many users with disabilities — especially invisible ones like dyslexia, low vision, or cognitive differences — hide their needs at work or school rather than formally disclose. The ask itself carries social and professional cost.

AccessBridge does: zero disclosure. The user never tells the website, the employer, or a coworker. Every adaptation happens silently on the user's own browser; the website never knows an accommodation fired.

Engages · Ambient Detection · On-Device Processing · Privacy Guarantees
Access

Assistive tools cost more than the device

Commercial screen readers run $900 – $1,200. Eye trackers start at $150 and climb past $25,000 for medical-grade. Enterprise assistive suites license at $400/user/year. Most people who need them can't afford them.

AccessBridge does: ships as a free Chrome extension. Runs on any laptop made in the last five years. Uses the webcam you already have, the microphone you already have, and the browser you're already in.

Engages · Everything — every feature is free and local

Privacy & Performance Guarantees

Non-negotiables the architecture was designed around from day one.

  • 100% on-device for behavioral data — zero network I/O for struggle scoring or adaptation decisions.
  • Zero disclosure — the user never tells a website they need help; the website never knows an adaptation fired.
  • Reversible by default — every adaptation carries a revert handle. Master toggle off = every mutation undone in one message.
  • AI cascade — free local tier first; paid tiers gated by a daily cost budget tracked in-browser.
  • Strict TypeScript end-to-end; no any, no // @ts-ignore in production code.
  • Adaptation latency < 50 ms from signal threshold crossing to DOM change.
  • Content-script syntax-check runs on every build (node -c dist/…/content/index.js) to catch the chunk-collision trap before it ships.
  • 187 unit tests cover struggle scoring, decision rules, profile persistence, transliteration, language detection, and the AI cache / cost tracker.

Beyond the extension

The Full AccessBridge Vision

The browser extension is the wedge. The real platform is the core + AI engine — every surface where accessibility gaps exist is a candidate deliverable.

Tier 1

Same Core, New Surfaces

Everywhere accessibility matters — not just browsers.

  • Desktop companion (Tauri) — ambient adaptation for Word, Excel, Teams, Slack, VS Code
  • Android accessibility service — system-wide for native apps
  • iOS app — Safari extension + in-app SDK
  • CLI / Terminal mode — jargon decoder, error-message simplifier
Tier 2

Enterprise Layer

Deploy + monitor accessibility across an organization.

  • Admin console — org-wide profile defaults, aggregate struggle heatmaps
  • Compliance scanner — WCAG / Section 508 audits with reports
  • Profile sync + SSO — SAML/OIDC, profile follows the user
  • Audit log + ROI reporting — which apps cause most struggle
Tier 3

Ecosystem (Developer)

Turn the AI engine into a platform, not just a product.

  • JS/TS SDK — embed AccessBridge in any web app, no user install
  • Public API — summarize + simplify endpoints for integrations
  • Figma plugin — audit designs for accessibility before build
  • VS Code extension — warn devs writing inaccessible markup
  • Domain connector framework — community contribs per industry
Tier 4

Research / Moonshot

Where accessibility intersects with ambient computing.

  • Real-time meeting captions + translation — Teams, Zoom, Meet
  • AR overlay for smart glasses — menu reading, sign translation
  • Voice-first OS layer — full voice navigation for Windows
  • On-device ML models — WASM + WebGPU, zero cloud dependency
Strategic take: for the ideathon, the extension alone demonstrates the full architecture. For post-ideathon: Desktop app (Tauri) unlocks 80% of work computing outside the browser; Enterprise admin console opens B2B revenue; Public SDK + API turns the AI engine into a platform. The existing monorepo (core + ai-engine + extension) is already structured so every new surface is just another consumer of the same packages.
