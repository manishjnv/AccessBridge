Run a comprehensive secrets scan on the AccessBridge repo.

Install gitleaks (prefer: go install github.com/gitleaks/gitleaks/v8@latest — go is available; fallback: download GitHub release binary for Windows amd64; fallback to detect-secrets via pip install detect-secrets). Also install trufflehog if possible (pip install truffleHog3 or download binary).

Run two passes:

PASS 1 — Working tree (including uncommitted and untracked files):
- gitleaks detect --source . --no-git --report-format json --report-path /tmp/gitleaks-worktree.json --redact
- (or equivalent with detect-secrets scan > /tmp/detect-secrets-worktree.json)

PASS 2 — Full git history (all commits, all branches):
- gitleaks detect --source . --log-opts="--all" --report-format json --report-path /tmp/gitleaks-history.json --redact
- (or: detect-secrets scan --all-files with git log parsing)

Look for:
- AWS access keys: AKIA[0-9A-Z]{16}, ASIA[0-9A-Z]{16}
- Anthropic/OpenAI: sk-[A-Za-z0-9]{20,}
- xAI: xai-[A-Za-z0-9]{20,}
- HuggingFace: hf_[A-Za-z0-9]{30,}
- GitHub: ghp_, gho_, ghu_, ghs_, ghr_, github_pat_
- Google API: AIza[A-Za-z0-9_-]{30,}
- Google OAuth: GOCSPX-[A-Za-z0-9_-]{20,}
- Slack: xox[bposra]-[A-Za-z0-9-]{10,}
- Generic: password/secret/token/api_key assignments with 8+ char literal values (NOT URLs, NOT CSS gradients)
- Private keys: -----BEGIN .* PRIVATE KEY-----
- JWT tokens: eyJ[A-Za-z0-9_-]{10,}\.eyJ[A-Za-z0-9_-]{10,}
- Database URLs with embedded credentials: postgres://user:password@host/db, mysql://, mongodb://
- PEM / PFX files: *.pem, *.key, *.pfx, *.p12 checked in
- .env, .envrc, .env.local, credentials.json, gcp-service-account*.json checked in

Secondary scan:
- Base64 strings that decode to secret-looking material (32+ byte blobs)
- Hex strings of length 64 that aren't known to be hashes
- Hardcoded test fixture passphrases that could be real

For every finding classify:
- True positive real secret → CRITICAL. Must rotate immediately; add to report top.
- Test fixture / obviously fake → INFO with justification
- Uncertain → HIGH with "verify and rotate if real" action

Output: docs/security/secrets-scan.md

Format:

## Scan methodology
(tool, version, scope, flags)

## Pass 1 — Working tree
### Findings
(one entry per finding: file:line, rule, redacted match, classification, action)

## Pass 2 — Git history
### Findings
(include commit SHA + path for each)

## Summary
(tool used, files + commits scanned, findings by severity)

If any real secret is found, list it FIRST as a CRITICAL finding with explicit "rotate now" instruction. Do NOT modify source files.
